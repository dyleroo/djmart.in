UMBRA

  • A 2D puzzle platformer prototype
  • Explore a dark and foreboding forest, serving as a metaphor for depression, and attempt to escape from the dark envionment and find the light to escape
  • Avoid traps, monsters and other dangers to navigate the 4 levels
  • Coded in Unity using C# scripting
  • Video Trailer Coming Soon

ENTER THE PACVERSE

  • A multi-level adventure where Pac-Man is able to travel via portals to 3 different mini-games
  • Mini-games include a procedurally generated vertical platformer where Pac-Man must jump from platform to platform while avoiding falling to his demise. Features powerups, ghost enemy characters and moving platforms.
  • A 3D endless runner that spawns lines of enemies with a small gap the player must navigate to. Distance is tracked as the highscore.
  • A Frogger style arcade game where Pac-Man must cross multiple roads being patrolled by ghosts, trying to collect fruit on the other side to gain points.

A* PATHFINDING TECH DEMO

  • A 2D top down exploration game. Search for keys within a cave to unlock portals to the next level and collect three keys to win.
  • Companion character that will follow the player and battle enemies
  • Enemy character that utilises A* pathfinding and state machine to engage player and companion
  • Health tracker that is affected by damage and health cubes
  • Source Code (coming soon)

WE MEAN MOVIES

  • A MEAN Stack web app allowing you to browse, search and leave reviews on the top 100 movies IMDb
  • Source Code

BLOGAMING

  • A Ruby on Rails application that presents a discussion board where users can communicate asynchronously on a variety of topics
  • The discussion threads are maintained in a SQLite database, with a web interface that allows users to contribute to an existing topic or to create a new topic for discussion
  • Source Code

WEATHERWATCHER

  • WeatherWatcher is a python application that allows a weather presenter to choose locations, days and times from a stored list and then parses the 5-day forecast data retrieved from the OpenWeatherMap API to generate a series of images that can be used in a weather forecast TV broadcast
  • Users can also add additional locations to those supported by providing the latitude and longitude of the location as well as a label for display on the map
  • Location and day/time information should be stored in an SQLite database so that the same presentation can be easily generated again in the future
  • Source Code
